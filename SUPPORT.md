# Support

## Getting Help

Use the following channels based on the request type:

- Bugs: open a bug report using the GitHub issue template.
- Feature proposals: open a feature request with clear motivation and acceptance criteria.
- Questions: open a question issue with environment details and what you have already tried.
- Security concerns: follow [SECURITY.md](SECURITY.md).

Project maintainer: [Aditya K. R. Pand](https://github.com/Adityakrpand).

## Before Opening An Issue

Please check:

- [README.md](README.md)
- [docs/11_User_Guide/README.md](docs/11_User_Guide/README.md)
- [docs/12_Developer_Guide/README.md](docs/12_Developer_Guide/README.md)
- [docs/07_Testing/README.md](docs/07_Testing/README.md)
- Existing issues and pull requests.

## Useful Environment Details

Include:

- Operating system and version.
- Python version.
- Node.js and npm versions.
- Browser.
- PX4 SITL status and MAVSDK address.
- Backend and frontend logs.
- Exact command that failed.

## Support Boundary

Version 1.0 support covers local development, backend/frontend testing, mission storage behavior, PX4 SITL workflow verification, and project documentation. Physical flight support is outside the current release scope.
