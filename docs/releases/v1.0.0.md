# Vimantra AI Mission Planner v1.0.0

Release date: 2026-07-19

## Features

- Interactive React, TypeScript, Leaflet, and Tailwind mission-planning interface.
- SQLite mission and waypoint persistence.
- Mission validation, readiness scoring, deterministic analytics, and pre-flight checks.
- MAVSDK connection to PX4 SITL, mission upload, arm, disarm, mission start, and telemetry monitoring.
- Standard API envelopes, correlated request IDs, environment validation, structured logs, and audit events.
- Repeatable backend-only and connected PX4 SITL verification workflows.

## Architecture Summary

The React frontend manages operator interaction and calls a FastAPI backend through typed API services. FastAPI coordinates mission storage, independent validation and analytics services, pre-flight checks, and a single MAVSDK vehicle connection. SQLite stores missions locally. MAVSDK communicates with a separately running PX4 SITL instance over the configured system address.

## Supported Workflow

1. Create or load a waypoint mission.
2. Validate the mission and review readiness analytics.
3. Connect the backend to PX4 SITL.
4. Confirm telemetry and run pre-flight checks.
5. Upload the mission.
6. Arm the simulated vehicle and start the mission.
7. Monitor telemetry and mission progress until completion.
8. Disarm the simulated vehicle.

The connected workflow can be checked with `simulation/verify_px4_workflow.ps1`. The backend-only path uses the `-SkipDroneActions` switch.

## Known Limitations

- Version 1.0 supports PX4 SITL and local simulation; it is not certified for physical-aircraft operation.
- Autonomous AI mission generation is not included and remains planned for Version 2.0.
- The backend maintains one in-memory drone session and does not support fleets or multiple operators.
- Mission records can be created, listed, loaded, and deleted; editing an existing database record is not supported.
- Map markers visualize the route but do not support click-to-place or drag-to-move interaction.
- Full flight verification requires an independently running PX4 SITL instance reachable by MAVSDK.
