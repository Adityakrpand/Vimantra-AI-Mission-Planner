# API Response Standard

Sprint 14 standardizes all backend API responses for the frontend and future AI services.

## Response Envelope

Every JSON response uses the same envelope:

```json
{
  "success": true,
  "request_id": "7f4c3d0a-7ed7-45e7-93ab-b929cd43bc24",
  "data": {},
  "error": null
}
```

Error responses use the same shape:

```json
{
  "success": false,
  "request_id": "7f4c3d0a-7ed7-45e7-93ab-b929cd43bc24",
  "data": null,
  "error": {
    "code": "NOT_FOUND",
    "message": "Mission 404 was not found.",
    "details": null
  }
}
```

## Fields

- `success`: `true` for successful responses and `false` for errors.
- `request_id`: unique request identifier, generated by the API or accepted from `x-request-id`.
- `data`: endpoint payload for successful responses.
- `error`: structured error payload for failed responses.

## Error Categories

- `VALIDATION_ERROR`: malformed request body, invalid path/query data, or schema validation failure.
- `MISSION_INVALID`: mission validation failed during a blocking workflow such as upload.
- `BAD_REQUEST`: invalid business request.
- `NOT_FOUND`: requested resource does not exist.
- `CONFLICT`: request conflicts with current system state, such as drone actions while disconnected.
- `INTERNAL_SERVER_ERROR`: unexpected server failure.

Mission validation details keep domain error codes such as:

- `MISSION_NAME_REQUIRED`
- `ALTITUDE_TOO_LOW`
- `ALTITUDE_TOO_HIGH`
- `INVALID_LATITUDE`
- `INVALID_LONGITUDE`
- `SPEED_TOO_LOW`
- `SPEED_TOO_HIGH`
- `DUPLICATE_WAYPOINT`

## Request IDs

Clients may send:

```text
x-request-id: client-generated-id
```

If omitted, the backend generates one. The request ID is returned in both the response body and the `x-request-id` response header, and it is included in structured backend logs.

## HTTP Status Codes

The API uses:

- `200 OK`: successful read/action responses.
- `201 Created`: mission created.
- `400 Bad Request`: mission upload rejected by validation or invalid business request.
- `404 Not Found`: missing routes or resources.
- `409 Conflict`: drone state conflict.
- `422 Validation Error`: request schema validation failure.
- `500 Internal Server Error`: unexpected system error.

Python stack traces are logged internally and are never returned to API clients.
